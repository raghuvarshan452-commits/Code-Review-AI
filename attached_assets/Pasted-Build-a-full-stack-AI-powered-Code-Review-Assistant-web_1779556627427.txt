---

## FEATURE 2: FETCH PR DIFF — /api/fetch-pr

POST route that accepts { owner, repo, prNumber }

Steps:
1. Fetch PR metadata: GET https://api.github.com/repos/{owner}/{repo}/pulls/{prNumber}
   Headers: Authorization: Bearer GITHUB_TOKEN, Accept: application/vnd.github.v3+json
   
2. Fetch PR diff: GET https://api.github.com/repos/{owner}/{repo}/pulls/{prNumber}/files
   This returns array of files with filename, patch (the diff), additions, deletions, status

3. Return: { title, author, branch, filesChanged, additions, deletions, files: [{filename, patch, status, additions, deletions}] }

---

## FEATURE 3: AI REVIEW ENGINE — /api/review

POST route that accepts { files } and streams the response using ReadableStream.

Send to Claude API with this EXACT system prompt:

"""
You are an elite code reviewer with expertise in security vulnerabilities, performance optimization, and software engineering best practices.

Analyze the provided PR diff and return a JSON array of review comments. Each comment must follow this exact structure:

{
  "id": "unique string id",
  "severity": "critical" | "warning" | "suggestion",
  "category": "security" | "bug" | "performance" | "code-quality" | "best-practice",
  "file": "filename string",
  "line": line number as integer,
  "title": "short 5-8 word title of the issue",
  "issue": "1-2 sentence description of exactly what is wrong",
  "fix": "the corrected code snippet only — no explanation, just working code",
  "explanation": "2-3 sentences explaining why this matters and what the fix achieves",
  "cwe": "CWE-XXX if security issue, else null"
}

Security checks (ALWAYS scan for these):
- SQL injection, NoSQL injection
- XSS and HTML injection  
- Hardcoded secrets, API keys, passwords
- Insecure authentication or authorization
- Path traversal, command injection
- Insecure dependencies or imports
- OWASP Top 10 patterns

Bug checks:
- Null/undefined dereference
- Off-by-one errors
- Async/await mistakes, unhandled promises
- Memory leaks
- Incorrect error handling

Performance checks:
- N+1 queries
- Inefficient loops or algorithms
- Missing indexes implied by query patterns
- Unnecessary re-renders (React)
- Large bundle imports

Return ONLY a valid JSON array. No markdown fences. No explanation outside the JSON. If no issues found, return an empty array [].
"""

User message: "Review this PR diff:\n\n" + concatenated file diffs

Stream the response back to the client.

## FEATURE 5: POST COMMENTS TO GITHUB — /api/post-comment

POST route accepting { owner, repo, prNumber, comments }

For each comment, call:
POST https://api.github.com/repos/{owner}/{repo}/pulls/{prNumber}/reviews

Body:
{
  "body": "**[ReviewBot AI] ${comment.title}**\n\n${comment.issue}\n\n**Suggested fix:**\n\`\`\`\n${comment.fix}\n\`\`\`\n\n${comment.explanation}${comment.cwe ? '\n\n> ' + comment.cwe : ''}",
  "event": "COMMENT"
}

Return { success: true, posted: count }

## IMPORTANT IMPLEMENTATION NOTES

1. The review API route must use streaming. Use ReadableStream and return it directly. On the frontend, use fetch() with response.body reader to stream the JSON as it arrives.

2. Parse the streamed JSON carefully — Claude will return the full JSON array at once at the end of the stream, so buffer the entire stream then parse.

3. For syntax highlighting in fix panels, detect language from file extension:
   .js/.jsx → javascript, .ts/.tsx → typescript, .py → python, .go → go, .java → java, .rb → ruby, .php → php, default → plaintext

4. GitHub API rate limit: 60 req/hr unauthenticated, 5000/hr authenticated. Always send the Authorization header.

5. Store review history in localStorage as array of { id, prUrl, title, timestamp, issueCount, data }. Max 20 items.

6. All API errors should return { error: string } with appropriate HTTP status. Show error toast in UI.

7. The "Post All to GitHub" creates a single PR review with all comments batched, not individual API calls per comment.

8. Inter font: add to globals.css as @import url('https://fonts.googleapis.com/css2?family=Inter:wght@400;500;600&display=swap') and set font-family: 'Inter', sans-serif on body.
