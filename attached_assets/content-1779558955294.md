**Describe the bug**

In an Electron based app, we're using [HttpsProxyAgent](https://github.com/TooTallNate/node-https-proxy-agent) v4.0.0 with Axios to implement proxy mechanism. It used to work on Axios v0.19.0, but right now failed after upgraded to v0.19.2. No network data is re-directed to the proxy as defined.

All the codes and configurations are the same, except the Axios version upgrade. We found it seems the proxy settings are ignored by Axios, no matter the settings are correct or not. We've also tested against both http proxy and https proxy, with the same results.

**To Reproduce**

```
    const agent = new HttpsProxyAgent(
      {
        host: store.state.Settings.connection.proxy.host,
        port: store.state.Settings.connection.proxy.port,
        auth: `${store.state.Settings.connection.proxy.username}:${store.state.Settings.connection.proxy.password}`,
        rejectUnauthorized: false
      }
    )
    options.httpAgent = agent
    options.httpsAgent = agent

    const response = await axios(options)
```

**Expected behavior**

HttpsProxyAgent can work as before.

**Environment:**

- Axios Version: 0.19.2
- OS: Windows 10 & macOS 10.15
- Electon: 8.2.2
